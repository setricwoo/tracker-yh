# 央行表态 + FedWatch — 完整更新流程

> 本文档描述央行官员表态和CME FedWatch利率预期的完整更新流程。AI每次执行时按步骤操作即可。

---

## 流程概览

```
Step 1: 运行 jin10_fetch.py 抓取金十快讯
    ↓
Step 2: AI读取数据，识别→确认→去重→总结
    ↓
Step 3: 与已有数据合并，输出完整JSON到 cb-statements.json
    ↓
Step 4: 运行 fetch_fedwatch.py 抓取CME FedWatch利率预期
    ↓
Step 5: 自动保存到 fedwatch.json（脚本自动完成，无需AI介入）
```

---

## Step 1：抓取金十数据

运行以下命令（在项目根目录下）：

```bash
python scripts/jin10_fetch.py
```

- 默认增量模式：读取已有 `data/jin10_cb_for_ai.json` 最新时间戳，仅抓取更新的快讯

输出文件：`data/jin10_cb_for_ai.json`

---

## Step 2：AI处理数据

读取 `data/jin10_cb_for_ai.json` 中的 items，完成四个步骤：

### 步骤一：识别人名

从快讯中提取所有被提及的人物名字（中文名、英文名、日文名等）。

**注意翻译差异**，同一个人可能有不同译名：
- 如 "Goolsbee" / "古尔斯比" 是同一人
- "Villeroy de Galhau" / "维勒鲁瓦" / "维勒鲁瓦·德加洛" 是同一人
- "植田和男" / "植田" / "Ueda" 是同一人
- "冰见野良三" / "冰见野" / "Himino" 是同一人

### 步骤二：确认是否为央行货币政策委员会成员

对识别出的每个人名，判断其是否属于以下机构的**现任货币政策委员会成员**：

| 央行 | 委员会名称 | 确认方法 |
|------|-----------|---------|
| 美联储 | FOMC | 理事会7人 + 12家地区联储主席（其中5人轮流投票） |
| 欧央行 | 管委会 | 6人执委 + 20国央行行长 |
| 日本央行 | 政策委员会 | 9人（行长+2副行长+6审议委员） |
| 英格兰银行 | MPC | 9人（行长+3副行长+首席经济学家+4外部委员） |
| 中国人民银行 | 货币政策委员会 | 13人（行长+副行长+各部委负责人+专家委员） |
| 瑞士央行 | 理事会 | 3人 |
| 加拿大央行 | 理事会 | 约6人 |
| 澳大利亚央行 | 理事会 | 约9人 |
| 新西兰联储 | 委员会 | 约5人 |
| 其他 | — | 凡是央行行长/副行长/理事，均应纳入 |

**判断标准**：
- 只要此人**当前是**某央行货币政策决策机构的成员，就纳入
- 行长、副行长、理事、执委、审议委员、外部委员都算
- 如果不确定此人身份，请自行搜索确认
- **排除**：已离任的前官员、分析师、经济学家（非委员）、记者

### 步骤三：去重合并

**同一位官员在1-2天内的多条快讯，大概率是同一次发言/采访。** 合并规则：

1. **同一官员** + **同一天或相邻一天** + **主题相关** = 同一场发言
2. 合并后保留最早时间戳
3. 标注合并了几条原始快讯

### 步骤四：总结提炼

对每条合并后的发言，按**话题**归纳总结为知识点。要求：

1. **按话题分组**：同一官员的多次发言可能涉及多个话题（如通胀、就业、能源冲击、金融监管等），将相关知识点归纳在一起，用编号列出
2. **适当提炼**：不要逐条快讯原文堆砌，而是综合归纳。例如10条快讯中有5条都在讲通胀，就合并为1-2条精炼的通胀要点，而不是列出5条重复或近似的内容
3. **保留关键细节**：具体数据、明确表态、政策暗示等不要丢失
4. **不同话题分开列点**：每个知识点一个完整句子，逻辑上互不重叠
5. 不需要额外写摘要或标注立场

**示例**：

原始快讯（假设合并了8条）：
- 鲍威尔说通胀预期稳固
- 鲍威尔说能源冲击通常短暂
- 鲍威尔说关税推升通胀0.5-1个百分点
- 鲍威尔说关注通胀预期变化
- 鲍威尔说就业增长非常缓慢
- 鲍威尔说劳动力市场面临长期性挑战
- 鲍威尔说美联储需要保持独立
- 鲍威尔说AI正在提高生产力

总结后points：
1. **通胀**：关税将推升通胀0.5至1个百分点，能源冲击通常短暂、标准应对是耐心等待其消退，长期通胀预期仍保持稳定，但需密切关注通胀预期变化
2. **就业**：当前就业增长非常缓慢，劳动力市场面临长期性趋势性挑战，但两大目标之间存在紧张关系
3. **政策立场**：当前政策处于有利地位可等待观察，美联储需保持独立性，无意与任何政府或政治家作对
4. **经济前景**：对中长期前景非常乐观，人工智能正在提高生产力

## Step 3：与已有数据合并并输出

### 3.1 读取已有数据

读取 `data/cb-statements.json`，获取已有条目。

### 3.2 合并规则

1. **新增**：新识别的官员发言，按日期倒序插入
2. **更新**：同一官员+同一天已有记录，合并知识点（去重）
3. **保留**：已有数据中未被覆盖的条目全部保留
4. **最终排序**：所有条目按日期倒序排列

### 3.3 输出格式

**直接输出JSON到 `data/cb-statements.json`，不要输出其他内容。**

```json
{
  "fetchTime": "2026-04-03T12:00:00",
  "totalItems": 50,
  "items": [
    {
      "date": "2026-04-03",
      "bank": "美联储",
      "official": "鲍威尔（Jerome Powell，主席）",
      "title": "美联储主席",
      "mergedCount": 3,
      "points": [
        "知识点1",
        "知识点2",
        "知识点3"
      ],
      "link": "https://www.jin10.com/flash/原始快讯ID"
    },
    {
      "date": "2026-04-03",
      "bank": "美联储",
      "official": "古尔斯比（Austin Goolsbee，芝加哥联储主席）",
      "title": "芝加哥联储主席",
      "mergedCount": 9,
      "points": [
        "知识点1",
        "知识点2"
      ],
      "link": "https://www.jin10.com/flash/原始快讯ID"
    }
  ]
}
```

**字段说明**：
- `fetchTime`：本次更新时间（ISO格式）
- `totalItems`：合并后总条目数
- `date`：发言日期（YYYY-MM-DD）
- `bank`：央行中文名（如"美联储"、"欧央行"、"日本央行"、"英格兰银行"、"中国人民银行"等）
- `official`：中文名（英文名，职位），如"鲍威尔（Jerome Powell，主席）"
- `title`：职位简称，如"主席"、"芝加哥联储主席"
- `mergedCount`：本次合并了几条原始快讯
- `points`：知识点列表，每个都是完整的一句话
- `link`：最早那条快讯的链接

---

## Step 4：更新 CME FedWatch 利率预期

完成Step 3后，运行以下命令抓取CME FedWatch最新概率数据：

```bash
python scripts/fetch_fedwatch.py
```

**脚本工作原理**：
1. 用Playwright打开CME FedWatch Tool页面
2. 自动找到页面内的QuikStrike iframe（实际的交互工具）
3. 在iframe左侧导航中点击"Probabilities"
4. 提取多会议概率矩阵表格
5. 自动保存到 `data/fedwatch.json`

**无需AI介入**：脚本自动完成数据提取和保存。

**注意事项**：
- 脚本使用非headless模式（会弹出浏览器窗口），不要手动操作该窗口
- 页面加载可能需要1-2分钟，耐心等待
- 如果超时失败，可重新运行一次
- 输出文件：`data/fedwatch.json`，包含14个FOMC会议的概率数据

---

## 注意事项

1. **只保留央行货币政策委员会成员本人的发言**，过滤分析师/市场评论
2. **总结要精炼**：同话题合并提炼，不要逐条堆砌原文；但关键数据、明确表态、政策暗示等细节必须保留
3. **按话题组织**：每个point聚焦一个话题（如通胀、就业、利率、能源冲击、金融稳定等），一个point内可以包含该话题的多个层面
4. 时区为北京时间（UTC+8）
5. 同一条快讯中提到多位官员时，分别记录
6. **只输出JSON到 cb-statements.json**，不要输出markdown或其他格式
7. 网页 `central-bank-tracker.html` 会自动读取 `data/cb-statements.json` 和 `data/fedwatch.json` 渲染
